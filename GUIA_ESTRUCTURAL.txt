                    GUIA ESTRUCTURAL DEL PROYECTO ELMASME
                         (Universal Viewer / Visor Universal)
================================================================================

Fecha de generacion: Abril 2026
Proposito de este archivo: Servir como referencia rapida para agentes de codigo
y desarrolladores que necesiten comprender la arquitectura del proyecto sin tener
que leer todos los archivos fuente.

================================================================================
SECCION 1 - EXPLICACION TECNICA Y SINTETICA DEL PROYECTO
================================================================================

ELMASME es una aplicacion de escritorio multiplataforma construida con Python y
PySide6 (bindings de Qt 6) que funciona como un visor universal de archivos. Su
proposito es abrir, visualizar, editar y convertir multiples tipos de archivos
multimedia y documentos desde una unica interfaz.

TECNOLOGIAS BASE:
- Lenguaje: Python 3
- Framework GUI: PySide6 (Qt 6)
- Conversion multimedia: FFmpeg / FFprobe (dependencia externa)
- Lectura PDF: QPdfDocument / QPdfView (integrado en PySide6)
- Lectura DOCX: zipfile + xml.etree.ElementTree (libreria estandar Python)
- Integracion Windows: ctypes + winreg (registro del sistema)

ARQUITECTURA GENERAL:
La aplicacion sigue un patron de enrutamiento por tipo de contenido. El flujo es:

  1. main.py inicia la aplicacion Qt y crea la ventana principal.
  2. app_window.py gestiona la interfaz con pestanas (tabs), navegacion entre
     archivos, menu de configuracion, temas y botones del footer.
  3. Cuando se abre un archivo o carpeta, file_navigation.py construye una lista
     de archivos compatibles en el directorio y permite navegar secuencialmente.
  4. content_viewers.py recibe la ruta del archivo, consulta formats.py para
     determinar el tipo (imagen, audio, video, pdf, texto, docx) y enruta al
     visor especializado correspondiente usando un QStackedWidget.
  5. Cada tipo de contenido tiene su propio conjunto de modulos:
     - Imagen: image_viewer.py + image_converter.py
     - Audio: audio_player.py + audio_converter.py + audio_playlist.py
     - Video: video_player.py + video_converter.py + video_playlist.py
     - Documento: document_viewer.py + document_editor.py + document_pdf.py
  6. Los modulos *_handler.py son capas de re-exportacion que agrupan las
     importaciones de cada subsistema para simplificar los imports.

PATRON DE MODULOS:
Cada tipo de archivo sigue la estructura:
  - *_handler.py  -> Modulo agregador (re-exporta clases y constantes)
  - *_viewer.py o *_player.py -> Interfaz grafica del visor/reproductor
  - *_converter.py -> Logica de conversion de formatos
  - *_playlist.py  -> Gestion de listas de reproduccion (solo audio y video)

SISTEMA DE TEMAS:
themes.py define 4 temas visuales (Oscuro, Claro, Cyberpunk 2038, Retro Terminal)
como cadenas QSS (Qt Style Sheets). Se aplican globalmente a toda la aplicacion.
El usuario los selecciona desde el dialogo de configuracion en app_window.py.

SISTEMA DE CONVERSION:
Las conversiones de audio y video dependen de FFmpeg. Usan subprocesos con
callbacks de progreso que se muestran en progress_bar.py (barra flotante).
Las conversiones de imagen usan QImage de PySide6 directamente.

INTEGRACION WINDOWS:
windows_integration.py maneja el registro de asociaciones de archivo en el
registro de Windows (HKEY_CURRENT_USER), permitiendo que el sistema operativo
abra archivos con ELMASME por defecto.

FORMATOS SOPORTADOS ACTUALMENTE:
  Imagen: .png, .jpg, .jpeg, .bmp, .gif, .webp, .tif, .tiff
  Audio:  .mp3, .wav, .flac, .ogg, .m4a, .aac, .wma
  Video:  .mp4, .mkv, .avi, .mov, .wmv, .webm, .m4v
  PDF:    .pdf
  DOCX:   .docx (solo lectura)
  Texto:  .txt, .py, .js, .json, .html, .css, .md, .xml, .yaml, .yml, .toml,
          .ini, .cfg, .conf, .log, .csv, .tsv, .sql, .sh, .bat, .ps1, .rb,
          .php, .java, .c, .cpp, .h, .hpp, .cs, .go, .rs, .ts, .jsx, .tsx,
          .vue, .svelte, .r, .m, .swift, .kt, .dart, .lua, .pl, .tex

SECCION 2 - MAPA ESTRUCTURAL DE ARCHIVOS
================================================================================

A continuacion se describe cada archivo, su responsabilidad y con que otros
archivos se relaciona. Use esta seccion para determinar que archivos necesita
leer o modificar segun la tarea asignada.

------------------------------------------------------------------------------
ARCHIVO: main.py
RESPONSABILIDAD: Punto de entrada de la aplicacion.
FUNCIONES CLAVE: resolve_start_path(), main()
DEPENDE DE: app_window.py
MODIFICAR SI: Necesita cambiar la forma en que se inicia la app, argumentos de
  linea de comandos, o configuracion inicial de QApplication.
------------------------------------------------------------------------------

------------------------------------------------------------------------------
ARCHIVO: app_window.py
RESPONSABILIDAD: Ventana principal, sistema de pestanas (tabs), footer con
  botones de navegacion, dialogo de configuracion, gestion de temas.
CLASE PRINCIPAL: UniversalViewerWindow(QMainWindow)
DEPENDE DE: content_viewers.py, file_navigation.py, formats.py, themes.py,
  windows_integration.py
MODIFICAR SI: Necesita cambiar la interfaz principal, agregar/quitar botones
  del footer, modificar el sistema de tabs, cambiar la navegacion entre archivos,
  modificar el dialogo de configuracion, o agregar nuevas funciones globales.
NOTAS: No tiene barra de menu superior (QMenuBar). Todo el acceso es por footer.
  El footer tiene: boton Archivo, navegacion (anterior/siguiente con contador),
  y boton de configuracion (engranaje). El tema se selecciona dentro del dialogo
  de configuracion.
------------------------------------------------------------------------------

------------------------------------------------------------------------------
ARCHIVO: formats.py
RESPONSABILIDAD: Identificacion centralizada de tipos de archivo.
FUNCIONES CLAVE: get_content_type(path), is_supported(path), display_type(path),
  supported_extensions(), get_file_category(path)
DEPENDE DE: Constantes de extensiones de todos los *_handler.py
MODIFICAR SI: Necesita agregar soporte para un nuevo tipo de archivo, cambiar
  la clasificacion de un formato, o modificar las etiquetas de tipo.
NOTAS: Es el HUB central de identificacion. Cualquier nuevo formato debe
  registrarse aqui y en el handler correspondiente.
------------------------------------------------------------------------------

------------------------------------------------------------------------------
ARCHIVO: file_navigation.py
RESPONSABILIDAD: Gestion de la lista de archivos y navegacion secuencial.
CLASE PRINCIPAL: FileNavigator
DEPENDE DE: formats.py (is_supported)
MODIFICAR SI: Necesita cambiar el orden de archivos, el filtrado, o la logica
  de navegacion anterior/siguiente.
------------------------------------------------------------------------------

------------------------------------------------------------------------------
ARCHIVO: themes.py
RESPONSABILIDAD: Definicion de temas visuales (hojas de estilo QSS).
FUNCION CLAVE: get_theme(name)
TEMAS DISPONIBLES: "Oscuro", "Claro", "Cyberpunk 2038", "Retro Terminal"
DEPENDE DE: Nada (modulo independiente)
MODIFICAR SI: Necesita agregar nuevos temas, cambiar colores/estilos de temas
  existentes, o agregar estilos para nuevos widgets.
NOTAS: El tema Oscuro usa base negra (#0a0a0a). Todos los temas incluyen estilos
  para QToolTip, QDialog, QMessageBox, QInputDialog.
------------------------------------------------------------------------------

------------------------------------------------------------------------------
ARCHIVO: content_viewers.py
RESPONSABILIDAD: Router de contenido. Usa QStackedWidget para mostrar el visor
  correcto segun el tipo de archivo.
CLASE PRINCIPAL: ViewerHost(QWidget)
DEPENDE DE: image_handler, audio_handler, video_handler, document_handler,
  formats.py
MODIFICAR SI: Necesita agregar un nuevo tipo de visor (ej: visor 3D, visor de
  hojas de calculo), cambiar la logica de enrutamiento, o agregar funciones
  transversales a todos los visores.
NOTAS: Contiene las instancias de ImageViewer, AudioViewer, VideoViewer,
  DocumentViewer y un QLabel para mensajes. Tambien gestiona operaciones de
  guardado de documentos.
------------------------------------------------------------------------------

------------------------------------------------------------------------------
ARCHIVO: image_viewer.py

FUNCIONES DESTACADAS: zoom_in/out, toggle_fullscreen, show_resize_dialog,

Las siguientes sugerencias proponen profundizar las opciones que ya existen en

   (nearest-neighbor, bilinear, bicubic, lanczos) en ImageResizeDialog para que

   Archivo: image_converter.py

   Archivos: image_converter.py, image_viewer.py

3. AUDIO - Agregar seleccion de bitrate especifico (64, 128, 192, 256, 320 kbps)
   y sample rate (22050, 44100, 48000 Hz) en AudioConverterDialog para control
   fino de la calidad de conversion.
   Archivo: audio_converter.py

4. AUDIO - Implementar visualizacion de forma de onda (waveform) o espectrograma
   basico en el reproductor de audio para retroalimentacion visual durante la
   reproduccion.
   Archivo: audio_player.py

5. VIDEO - Agregar seleccion de resolucion de salida (480p, 720p, 1080p, 4K) y
   framerate objetivo en VideoConverterDialog, ademas de los presets de calidad
   existentes.
   Archivo: video_converter.py

6. VIDEO - Implementar captura de fotograma (screenshot) del video en reproduccion,
   permitiendo guardar el frame actual como imagen en formato seleccionable.
   Archivo: video_player.py

7. DOCUMENTO - Agregar resaltado de sintaxis (syntax highlighting) para archivos
   de codigo fuente (.py, .js, .java, etc.) usando un lexer basico por
   extensiones, con coloreado de palabras clave, strings y comentarios.
   Archivos: document_viewer.py, document_editor.py

8. DOCUMENTO - Implementar contador de palabras, lineas y caracteres visible en
   la barra de estado del editor de texto, actualizandose en tiempo real.
   Archivos: document_viewer.py, document_editor.py

9. PLAYLIST - Agregar informacion de duracion total de la playlist y duracion
   individual de cada pista visible en la lista, extrayendo metadatos al agregar.
   Archivos: audio_playlist.py, video_playlist.py

10. PLAYLIST - Implementar busqueda/filtro dentro de la playlist para localizar
    rapidamente archivos por nombre cuando la lista es extensa.
    Archivos: audio_playlist.py, video_playlist.py


================================================================================
SECCION 4 - SUGERENCIAS DE NUEVOS TIPOS DE ARCHIVO A AGREGAR
================================================================================

Los siguientes tipos de archivo podrian ampliar la compatibilidad del visor:

1. SVG (.svg) - Graficos vectoriales escalables. Se pueden renderizar con
   QSvgWidget de PySide6. Agregaria al gestor de imagenes.
   Archivos a modificar: image_converter.py (extensiones), image_viewer.py
   (renderizado SVG), formats.py

2. EPUB (.epub) - Libros electronicos. Son archivos ZIP con HTML/CSS/imagenes.
   Se podrian extraer y renderizar en el visor de documentos.
   Archivos a modificar: document_viewer.py, formats.py, posible nuevo modulo

3. RTF (.rtf) - Formato de texto enriquecido. Qt tiene soporte parcial via
   QTextEdit. Se podria integrar en el visor de documentos.
   Archivos a modificar: document_viewer.py, document_editor.py, formats.py

4. HEIF/HEIC (.heif, .heic) - Formato de imagen moderno de Apple/Android.
   Requiere libreria pillow-heif o similar para decodificar.
   Archivos a modificar: image_converter.py, formats.py

5. AVIF (.avif) - Formato de imagen de nueva generacion basado en AV1.
   Soportado por Pillow 9.1+.
   Archivos a modificar: image_converter.py, formats.py

6. MIDI (.mid, .midi) - Archivos de musica MIDI. Se podrian reproducir con
   pygame.midi o similar, o convertir a audio con FluidSynth.
   Archivos a modificar: audio_converter.py, audio_player.py, formats.py

7. ODS/ODT (.ods, .odt) - Formatos de documento abierto (LibreOffice).
   Son archivos ZIP con XML, similar a DOCX.
   Archivos a modificar: document_viewer.py, formats.py

8. ICO (.ico) - Iconos de Windows. Qt los soporta nativamente via QImage.
   Archivos a modificar: image_converter.py, formats.py

9. FLV (.flv) - Flash Video. Soportado por FFmpeg para conversion y por
   Qt multimedia para reproduccion en algunos sistemas.
   Archivos a modificar: video_converter.py, formats.py

10. OPUS (.opus) - Codec de audio moderno de alta eficiencia. Soportado por
    FFmpeg y parcialmente por Qt multimedia.
    Archivos a modificar: audio_converter.py, formats.py


================================================================================
SECCION 5 - SUGERENCIAS DE NUEVAS FUNCIONES E IMPLEMENTACIONES
================================================================================

Las siguientes sugerencias son funciones nuevas que NO tienen relacion con
brillo, ecualizacion ni subtitulos:

1. SISTEMA DE MARCADORES (BOOKMARKS)
   Permitir al usuario marcar archivos como favoritos dentro de una carpeta.
   Los marcadores se guardarian en un JSON local y serian accesibles desde un
   panel lateral o boton dedicado para saltar rapidamente a archivos frecuentes.
   Archivos: app_window.py (UI), nuevo modulo bookmarks.py

2. HISTORIAL DE ARCHIVOS RECIENTES
   Mantener un registro de los ultimos N archivos abiertos, accesible desde el
   menu Archivo. Guardado en un JSON persistente en el directorio de usuario.
   Archivos: app_window.py, nuevo modulo recent_files.py

3. COMPARADOR LADO A LADO (SPLIT VIEW)
   Permitir abrir dos archivos (imagenes o documentos) en vista dividida para
   compararlos visualmente. Util para comparar versiones de un mismo archivo.
   Archivos: content_viewers.py, image_viewer.py, document_viewer.py

4. METADATOS EXIF/ID3 PANEL
   Panel desplegable que muestre los metadatos completos del archivo actual:
   EXIF para imagenes (camara, GPS, fecha), ID3 para audio (artista, album),
   y propiedades de video (codec, bitrate, resolucion) en un formato legible.
   Archivos: nuevo modulo metadata_panel.py, content_viewers.py

5. SISTEMA DE ETIQUETAS (TAGS) POR COLOR
   Permitir al usuario asignar etiquetas de color (rojo, verde, azul, etc.)
   a los archivos para clasificarlos. Las etiquetas se mostrarian como indicador
   visual junto al nombre del archivo y se podrian filtrar.
